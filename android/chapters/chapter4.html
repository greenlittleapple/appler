Recursion

Recursion is one of the more advanced topics of object-oriented programming (OOP), and this is one of the reasons that College Board chose Java/Karel as the AP Computer Science test language. Recursion, along with polymorphism (more on this later), are some of the best and useful features in OOP languages. Recursion is a way to repeat segments of code by calling a method within itself. As with any other iteration sequences in Java, recursion repeats the block of code until a certain condition is met. The first step to writing recursive code is to identify the situation in which nothing needs to be done, called the base case. The general case on the other hand, is when the stopping condition is not met and the code needs to recurse and iterate. An example of recursive code in Karel the Robot would be:
public void findBeeper() //we want to find a beeper ahead of us
{
If(nextToABeeper())
{
//do nothing. This is the base case.
}
else
{ // this is the general case
move();
findBeeper();
(calls the method again until the first if test is met)
}
}
By calling the method within itself, when the code is run and it reaches the point at which it gets called, it will go back to the beginning of the code and repeat it. This may seem confusing at first, but if you look over the code and try to see what happens then you will be able to grasp the topic.
Another topic within recursion is stacking. Stacking is when you add code beneath where the method is called within itself. For each time that the recursion point is reached, all the code beneath it will be run once when the stopping point is reached. So if we have the following code:
public void findBeeperThenMove() //find beeper then move
{
If(nextToABeeper())
{
//do nothing. This is the base case.
}
else
{ // this is the general case
move();
findBeeperThenMove();
move(); // *****THIS CODE IS BEING STACKED*****
}
}
The move(); statement after findBeeperThenMove(); is what is being stacked. If we have to move five times until we reach a beeper, as soon as we reach the beeper the robot will move five times. This is an example of stacking.
